# ReceiptHub v1.7.0 最終運用フロー

## 一般ユーザーの月選択

一般ユーザーは内部の提出サイクル月を意識せず、画面で「領収書発行月」を直接選択します。

```text
領収書発行月 2026年06月
  → ReceiptHub内部の提出サイクル 2026年07月
  → AIは2026年06月の領収書として日付確認
```

ユーザー画面では提出月、提出月の前月、ファイル保存期限を表示しません。

## サービス登録と例外申請

1. サービスマスターにあるサービスは「サービス利用登録」から直接登録します。
2. 一覧にない場合は、同じサービス利用登録画面内の青い「一覧にサービスがない場合」枠から例外申請へ進みます。
3. 例外申請はサービスマスターに存在しないサービスだけが対象です。
4. 例外申請画面には「サービス利用登録へ戻る」を常設します。
5. 承認されるまで、申請サービスは領収書アップロードの選択肢に入りません。

## 領収書アップロードと提出状態

- 選択した領収書発行月について、サービスごとに領収書を複数回・複数ファイルで追加できます。
- 従量課金 / APIで請求がなかった場合は「この月は利用なし」を選択します。
- 一部サービスが残っている状態は「未提出領収書あり」と表示します。
- 「提出する」を押した時に未提出サービスが残っていても、アップロード済み領収書は保持します。
- 黄色の注意メッセージで、残りの領収書またはAPI利用確認を忘れないよう案内します。
- 提出履歴には領収書発行月、サービス登録数、確認済み数、未提出・未確認数、領収書数を表示します。
- 提出日時と内部提出月は一般ユーザーの履歴には表示しません。

## AI確認と利用者名（宛名）

管理者がAI検査を実行すると、次の項目を確認します。

1. カード末尾4桁
2. 払先
3. 利用者名・宛名
4. サービスと払先の関連、または「その他」メモと領収書内容の関連
5. 日付
6. 金額
7. 通貨
8. 領収書発行月との一致

利用者名・宛名は `Billed to`、`Bill to`、`Customer`、`Account holder`、宛名、利用者名、購入者名などから抽出します。

- 対象ユーザーの登録氏名やメールアドレスと明確に一致: チェック済み
- 別人・別利用者と明確に判断: 未チェック + 管理者メモ
- 宛名なし、会社名・部署名だけ、表記が曖昧: 未チェック + 管理者確認

利用者名・宛名は独立した監査項目です。宛名だけが曖昧でも、払先・日付・金額・通貨等が揃っていればファイル名候補は保持します。

## 管理者確認

管理者の領収書確認画面では、AI抽出利用者名と確認メモを表示します。「利用者名（宛名）」を含む全8項目を人が確認し、全項目にチェックを入れた時だけ「管理者確認済み」にできます。

AIが宛名を確認できない領収書は削除せず、提出履歴でハイライトし、「利用者名（宛名）要確認」に集計します。

## 内部月次ルールと全社明細

内部では従来どおり、領収書発行月の翌月を提出・明細月として扱います。

```text
2026年07月分の全社ご利用代金明細
  → 2026年07月提出サイクル
  → 2026年06月発行分の全ユーザー領収書と照合
```
