# ReceiptHub v1.5.1 最終運用フロー

## ドラッグ＆ドロップ

次のファイル入力は、クリック選択に加えてドラッグ＆ドロップで登録できます。

- 一般ユーザーの領収書追加（複数ファイル可・選択後自動アップロード）
- 管理者の代理アップロード（複数ファイル可）
- 一般ユーザーおよび管理者の領収書ファイル差し替え
- 管理者の全社ご利用代金明細書アップロード

対応形式は PDF / PNG / JPG / JPEG / WEBP、1ファイル最大10MBです。サーバー側の拡張子・サイズ検証は従来どおり実行します。

ドラッグ中はアップロード枠の中央に「ここでファイルを離してください」を表示します。ドロップ後は受付件数とファイル名、送信開始後は「アップロード中」を明示し、色の変化だけに依存しません。

## 管理者代理アップロードから確認まで

1. 管理者は「提出履歴」から対象ユーザーの「月別状況」を開きます。
2. 対象サービスまたは「その他」を選び、領収書をクリック選択またはドラッグ＆ドロップで代理アップロードします。
3. 同じ月で別ユーザーの作業を続ける場合は、代理アップロード欄のユーザー選択、「前のユーザー」、「次のユーザー」を使用します。提出履歴へ戻る必要はありません。
4. アップロード済み領収書の「確認」を開きます。
5. PDF・画像の領収書本体を画面内で確認します。
6. AI未確認の場合だけ「AIでファイル名を修正・検査」を押します。処理済みの領収書は再検査しません。
7. AI結果を参考に、カード末尾7210、払先、サービス/払先関連、日付、金額、通貨、提出月一致を管理者がチェックします。
8. 表示・ダウンロード・ZIPで使用するファイル名を必要に応じて修正します。ストレージ上の内部保存名は変更しません。
9. すべての項目を確認できた場合は「すべて確認済みにする」を押します。確認者、確認日時、管理者メモを保存します。
10. 誤った領収書の場合は「再提出を指示」を押します。対象領収書を削除し、ユーザーへ再アップロード依頼を表示します。
11. 管理者自身が誤ったファイルを登録した場合は、確認画面でファイルを差し替え、AI確認を再実行します。

## AIと手動確認の関係

- AIは未確認の領収書だけを処理します。
- AI処理中は確認フォームを編集できず、完了後にリアルタイム更新します。
- AIが明確なカード不一致、サービス不一致、提出月不一致を検出した場合は、従来どおり自動取り下げ・再提出依頼を行います。
- AIが曖昧または読み取り不能と判断した項目は未チェックで残し、管理者が領収書本体を確認します。
- ファイル差し替え時はAI結果と管理者確認を未確認へ戻します。
