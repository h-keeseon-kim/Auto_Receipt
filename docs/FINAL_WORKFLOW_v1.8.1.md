# ReceiptHub v1.8.1 最終運用フロー

## Pカード利用設定の位置づけ

Pカード利用有無はサービスマスター共通の属性ではなく、**ユーザーごとの利用サービス**に保存します。同じサービスでも、利用者ごとにPカード支払いか、それ以外の支払いかを分けて管理できます。

```text
ユーザーA / ChatGPT（サブスク） / Pカード利用
ユーザーB / ChatGPT（サブスク） / Pカード未使用
```

Pカード利用サービスだけが、次の対象になります。

- 領収書アップロード
- 月次の必須提出・API利用確認
- 4日・10日のリマインドメール
- 全社ご利用代金明細との照合

Pカード未使用サービスは利用中サービスとして一覧に残りますが、領収書提出は不要です。 その月にPカード利用サービスも「その他」の領収書もない場合、月次ステータスは「提出不要」と表示します。

## 一般ユーザーのサービス登録

サービスマスターにあるサービスを「サービス利用登録」から追加する際、初期値は選択されていないため、次のどちらかを必ず明示的に選択します。

```text
Pカード利用
Pカード未使用
```

サービスマスターにない新規サービスを例外申請する場合も、サービス名、支払い方法、用途に加えてPカード利用有無を必須入力します。管理者が承認すると、その設定を対象ユーザーの利用サービスへ引き継ぎます。

## Pカード設定の後日変更

一般ユーザーは「利用サービス」の各行にある「Pカード設定」から、自分のサービス設定を変更できます。管理者は「利用サービス管理」→「ユーザー別サービス状況」→「編集」から変更できます。

```text
Pカード利用 → Pカード未使用
```

- 以後の領収書提出対象から除外
- リマインド判定から除外
- 全社Pカード明細のサービス候補から除外
- 開いている再提出依頼は対応済みに変更
- 過去の領収書ファイルと提出履歴は保持

```text
Pカード未使用 → Pカード利用
```

- 領収書提出・API利用確認の対象へ追加
- 未対応の場合はリマインド判定の対象
- 全社Pカード明細の照合候補へ追加

## 月次提出

月次サマリーはPカード利用サービスだけを提出対象として集計します。

```text
必須提出（サブスク等）
利用時提出（API等）
対応済みサービス数
未対応サービス数
```

Pカード未使用サービスは、別に「提出不要」として件数とサービス名を表示します。領収書アップロードのサービス選択肢にも表示しません。

従量課金 / APIの「この月は利用なし」申告も、Pカード利用サービスだけで利用できます。

## リマインドメール

リマインド対象は、利用中の一般ユーザーのうち、Pカード利用サービスに未提出・未確認項目があるユーザーだけです。

- 通常リマインダー：Pカード利用中の必須提出サービスに未提出がある場合
- 重要リマインダー：上記に加え、Pカード利用中のAPIサービスに利用確認待ちがある場合
- Pカード未使用サービスだけを利用しているユーザー：送信しない
- Pカード利用サービスの対応がすべて完了しているユーザー：送信しない

## 全社ご利用代金明細照合

全社ご利用代金明細はPカードの利用履歴であるため、候補サービスはPカード利用として登録されているユーザーサービスだけに限定します。

照合候補となる通常領収書は、アップロード時のPカード利用スナップショットが有効なものだけです。「その他」の追加領収書は、明細内容との一致候補として引き続き評価します。

サービスを後からPカード未使用へ変更しても、過去にPカード利用中としてアップロードした領収書の監査情報は保持します。

## 管理者画面

「利用サービス管理」→「ユーザー別サービス状況」で、次を確認できます。

- 利用中サービス数
- Pカード利用サービス数
- Pカード未使用サービス数
- 各サービスのPカード状態
- 登録元・停止元・最終領収書月
- Pカード設定を含むサービス編集

管理者の月次提出状況には、Pカード利用サービス数とPカード未使用・提出不要サービス数を分けて表示します。

## 既存データの移行

マイグレーション：

```text
receipts.0025_p_card_usage
```

移行時の初期値：

- 既存の全ユーザーサービス：Pカード利用
- 既存の全領収書スナップショット：Pカード利用
- 既存の例外申請：Pカード利用

既存の提出履歴、領収書ファイル、AI判定、再提出依頼、ご利用代金明細の解析結果は削除しません。

## サービスマスターからの利用ユーザー確認

管理者の「利用サービス管理」→「サービスマスター」では、各サービスの「利用中ユーザー」を展開すると、現在そのサービスを利用中のユーザー名とPカード利用状態を確認できます。

```text
OpenAI API（従量課金 / API）
├ user-a@example.com / Pカード利用
└ user-b@example.com / Pカード未使用
```

ユーザー名を選択すると「ユーザー別サービス状況」タブへ移動し、そのユーザーの全サービスを確認・編集できます。停止済みサービスの利用者は「利用中ユーザー」には含めず、従来の「割当合計」には履歴として含めます。利用者が多い場合は一覧内を縦スクロールします。

## ボタンラベルの表示ルール

日本語のボタン文字は途中で分割しません。画面幅が狭い場合は「サービス利用登 / 録」のように文字だけを折り返さず、ボタン全体を次の行へ移動します。この共通ルールを、通常ボタン、表内操作、ナビゲーション、管理タブ、ステータスバッジへ適用します。
