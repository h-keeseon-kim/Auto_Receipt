# ReceiptHub v1.1.1 最終運用フロー

## 月次業務

### 1. ユーザーのサービス確認

一般ユーザーはログイン後、「利用サービス」で自分の利用サービスを確認します。新規利用はサービスマスターから登録し、停止時は最終領収書月を指定します。

### 2. 領収書アップロード

対象月と登録サービスを選び、領収書をアップロードします。従量課金/APIを当月利用していない場合は「当月利用なし」を選択します。

返金、プラン変更、追加請求など、登録サービスに紐づかない予期せぬ領収書は「＋ その他の領収書」から追加します。項目は「その他」で固定し、内容メモと領収書ファイルを登録します。メモは必須です。提出済みの月へ追加した場合は、その月を下書きへ戻して再提出します。

### 3. ユーザー提出

全対象サービスについて、領収書または利用なし申告が揃うまで提出できません。

### 4. 管理者AI検査

管理者が提出履歴の「AIでファイル名を修正・検査」を押します。未処理項目だけが対象です。

- 明確な不一致: 自動取り下げ、再提出依頼
- 曖昧・読み取り不可: ファイルを保持し、手動確認対象
- 問題なし: ファイル名生成、チェック完了

その他領収書では必須メモをAIへ渡しますが、払先、日付、金額、通貨、カード番号など領収書ファイル内の情報を優先します。メモと領収書内容が明確に矛盾する場合は再提出対象、断定できない場合は管理者の手動確認対象とします。ファイル名には、領収書内容を中心にメモを補助情報として要約した短い取引ラベルを使用できます。

### 5. カード明細照合

管理者がユーザー別月次ページへご利用代金明細書をアップロードします。明細行を登録サービスと照合し、領収書が必要だが存在しない項目をハイライトします。

### 6. リマインダー

毎日1回 `send_receipt_reminders --kind auto` を実行します。

- 通常日: サブスク等の領収書が未提出の利用中ユーザー
- 警告日: サブスク等の未提出またはAPI利用確認待ちがある利用中ユーザー
- APIのみ未確認の場合、通常日は送らず警告日だけ送信
- 停止中ユーザーには送信しない

## 判定原則

- AIが明確に `false` と判定した不一致だけを自動取り下げ対象にします。
- AIが `unknown` / `null` / 確認不能とした項目は自動削除しません。
- カード明細の対象月は利用月ではなく請求・支払月を優先します。
- サービス名と払先は完全一致でなく、提供会社・請求名義の関連を評価します。
- 同一サービスの複数明細行には、領収書を1件ずつ割り当てます。

## スーパーアカウントの連絡先メール管理

スーパーアカウントの連絡先メールアドレスは任意です。「ユーザー管理」の「スーパーアカウント設定」で空欄にして保存できます。空欄にしてもログイン名、パスワード、スーパーアカウント権限は維持されます。解放したメールアドレスは、一般ユーザーまたは管理者ユーザーのアカウント名として登録できます。

CLIで空欄にする場合:

```bash
python manage.py clear_superuser_email --username admin
```
