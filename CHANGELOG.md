# Changelog

## 1.8.0 — 2026-07-23

ユーザーごとの利用サービスにPカード利用有無を追加し、領収書提出・リマインド・全社Pカード明細照合の対象をPカード利用サービスへ限定した運用拡張版です。

### Added

- 一般ユーザーのサービス利用登録で必須となる「Pカード利用 / Pカード未使用」選択
- 新規サービス例外申請で必須となるPカード利用有無
- 管理者によるユーザー別サービス登録・編集のPカード利用設定
- 一般ユーザーが利用中サービスのPカード設定を後から変更する画面
- Pカード提出対象がない月を「提出不要」と表示し、不要な提出操作を隠す月次ステータス
- ユーザー・管理者の利用サービス一覧にPカード状態バッジと件数集計
- 領収書のアップロード時点のPカード設定を保持する監査スナップショット
- ZIP `manifest.csv` の `p_card_usage_at_upload` 列

### Changed

- Pカード未使用サービスを領収書アップロード選択肢、月次必須提出、API利用確認から除外
- 新規登録フォームでPカード設定を自動選択せず、利用者・管理者による明示選択を必須化
- Pカード未使用サービスを4日・10日のリマインドメール判定から除外
- 全社ご利用代金明細のサービス候補・通常領収書候補をPカード利用分に限定
- Pカード未使用へ変更したサービスの未対応再提出依頼を対応済みに変更し、過去の領収書履歴は保持
- Pカード未使用サービスの利用停止では最終領収書月の入力を不要化
- ユーザー提出履歴と管理者月次状況に、Pカード利用対象数とPカード未使用・提出不要数を表示
- チュートリアルをPカード利用設定の説明へ更新
- 配布バージョンを `1.8.0` に更新

### Migration

- `receipts.0025_p_card_usage`
- 既存の全ユーザーサービスは `Pカード利用=True` として初期化
- 既存領収書の提出時Pカードスナップショットも `True` として初期化
- 既存の提出・領収書ファイル・AI結果・明細照合履歴は削除しません

## 1.7.2 — 2026-07-23

一般ユーザーの提出履歴を、サービス種別とサービス単位の進捗が一目で分かる表示へ整理したUX改善版です。

### Changed

- 「確認済みサービス数」を、管理者確認と混同しにくい「対応済みサービス数」へ変更
- 「未提出・未確認サービス数」を簡潔な「未対応サービス数」へ変更
- 提出履歴からファイル単位の「領収書数」を削除し、サービス単位の進捗へ統一
- 同じサービスへ複数ファイルを提出しても1サービスとして数えることを画面内に明記
- チュートリアルと運用ドキュメントを新しい提出履歴表示に更新
- 配布バージョンを `1.7.2` に更新

### Migration

- 既存の提出・領収書・サービスデータは変更しません

## 1.7.1 — 2026-07-23

AIの判定だけでは再提出依頼を出さず、管理者が領収書本体を確認して最終判断するワークフローへ修正し、既存の再提出依頼を管理画面から削除できるようにした運用修正版です。

### Added

- 領収書ごとの「AI再提出候補」と管理者向け候補理由

- 再提出依頼時にユーザーへ伝える任意理由の入力欄

- 表示月の再提出依頼一括削除
- 未対応・対応済みを含む全期間の再提出依頼一括削除
- ZIP `manifest.csv` のAI再提出候補・候補理由

### Changed

- 管理者が問題なしと判断して全項目を確認した場合、AI再提出候補の監査記録を残したまま「再提出判断待ち」を解消し、管理者確認済みにする
- 再提出依頼の文言を「再提出指示」から「再提出を依頼」へ統一
- 依頼削除後はユーザー画面の再提出通知も消える。過去に削除済みの領収書ファイルは復元しない
- 配布バージョンを `1.7.1` に更新

### Migration
