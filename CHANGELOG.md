# Changelog

## 1.3.0 — 2026-07-13

会社共通のご利用代金明細照合結果を、Slack共有に適したPDFとして出力できる機能追加版です。

### Added

- ご利用代金明細ごとの「照合結果PDF」ダウンロード
- 対象月、カード末尾、支払日、解析日時、最終照合日時をまとめたPDF概要
- 明細行数、領収書対象、確認済み、未提出、手動確認、対象外のPDF集計
- ユーザー別の未提出・要確認件数と関連サービスのサマリー
- 未提出行の黄色、手動確認行の薄橙色ハイライト
- 全明細照合結果の複数ページ出力とページ番号
- 日本語表示に対応したReportLabベースのPDF生成
- Slack共有先の権限確認を促す注意表示

### Changed

- AI解析中の明細では照合結果PDFボタンを表示せず、解析・照合完了後のみダウンロード可能
- PDF内の確認メモは共有しやすい長さに要約表示
- 配布バージョンを `1.3.0` に更新

### Migration

- なし

## 1.2.0 — 2026-07-12

会社共通のカード利用明細を全ユーザーの領収書と横断照合し、管理画面のサービス・ユーザー導線を整理した運用改善版です。

### Added

- 管理者専用の「ご利用代金明細」ページ
- 会社全体のカード明細と、対象月に全一般ユーザーが提出した領収書の一括照合
- 明細行ごとの照合ユーザー、対応サービス、対応領収書の表示
- 領収書未提出行の黄色ハイライトと、ユーザー・サービスが曖昧な行の薄橙色ハイライト
- 管理者による明細行のユーザー / サービス確定、対象外指定、最新領収書との再照合
- 全社明細の照合日時と未提出・手動確認件数
- 管理者チュートリアルの全社明細アップロード・照合説明

### Changed

- 「利用サービス管理」を「サービスマスター」「ユーザー別サービス状況」のタブ表示へ変更
- 「ユーザー管理」のスーパーアカウント設定をページ最下部へ移動
- 登録アカウント表の重複した「サービス」導線を削除
- カード明細をユーザー別データから会社共通データへ移行
- カード明細AIはユーザーを直接推測せず、サービスマスター候補と全明細行を抽出する方式へ変更
- 領収書照合は全ユーザー横断、一明細行につき一領収書の一対一割当へ変更
- ユーザー削除時も会社共通のカード明細は保持し、ユーザー紐付けのみ解除

### Migration

- `receipts.0016_global_card_statements`

## 1.1.1 — 2026-07-12

スーパーアカウントに設定した個人メールアドレスを安全に解放できる管理改善版です。

### Added

- 「ユーザー管理」内のスーパーアカウント連絡先メール設定
- スーパーアカウントのメールアドレスを空欄にする操作
- 緊急・運用時に使える `clear_superuser_email` 管理コマンド

### Changed

- スーパーアカウントの連絡先メール変更時、ログイン名とパスワードは維持
- 空欄にしたメールアドレスを一般ユーザーのアカウント名として再利用可能
- スーパーアカウントの連絡先メールを別アカウントが使用中の場合は重複を拒否

### Migration

- なし

## 1.1.0 — 2026-07-12

登録サービスに紐づかない予期せぬ領収書を、安全に追加提出できるようにした機能追加版です。

### Added

- アップロード画面の「＋ その他の領収書」導線
- 返金・プラン変更・追加請求など、登録サービス外の領収書アップロード
- 「その他」領収書で必須となる内容メモ
- 提出後に追加領収書を登録した場合の自動下書き戻し・再提出フロー
- その他領収書のメモを保持する再提出依頼
- チュートリアルへの追加領収書説明

### Changed

- OpenAI検査は、その他領収書の必須メモを参考情報として受け取る
- 払先・日付・金額・通貨・カード番号など、領収書ファイル内の情報をメモより優先
- その他領収書では「サービス/払先関連」の代わりに「メモ/領収書関連」を確認
- その他領収書のファイル名用ラベルは、領収書内容を中心にメモを補助情報として生成
- 管理者一覧、提出詳細、再提出待ち、ZIP manifestに追加領収書の区分とメモを反映

### Migration

- `receipts.0015_extra_receipts`

## 1.0.0 — 2026-07-12

ReceiptHubの最初の正式な業務フロー版です。

### Added

- 一般ユーザー / 管理者ユーザー / スーパーアカウントの権限分離
- スーパーアカウントによるユーザー権限変更
- ユーザー追加・削除・ステータス・パスワード管理
- サービスマスター、ユーザー別サービス、ユーザーによる利用登録・停止
- 月次領収書提出と従量課金/APIの「当月利用なし」申告
- 管理者手動実行によるOpenAI領収書ファイル名修正・検査
- 明確な不一致の自動取り下げと再提出依頼
- 曖昧な領収書の管理者レビュー用ハイライト
- カードご利用代金明細書のAI抽出、サービス・領収書照合、未提出行ハイライト
- 通常 / 重要リマインダーの対象判定
- 管理者編集可能な送信日、件名、本文
- Resend SMTPテスト送信と送信ログ
- 最大3ヶ月のファイル保存と期限切れ削除
- 初回ログインチュートリアル
- `VERSION` ファイルとバージョン付き配布フォルダー

### Changed

- アプリ名をReceiptHubへ統一
- 管理者の開始ページを提出履歴へ統一
- AI処理を定期Cronではなく管理者ボタンによる手動開始へ変更
- 同じサービスの複数カード明細行へ同一領収書を使い回さない照合方式へ変更

### Migration

- `receipts.0014_emailreminderschedule_initial_body_template_and_more`
